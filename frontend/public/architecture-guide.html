BITVERSE

Technical Architectural Reference & Deep Study Blueprint Guide

1. Project Vision & Purpose

BITVERSE is a bespoke digital library developed exclusively for First-Year engineering students at the Birla Institute of Technology (BIT) Mesra, Ranchi. It centralizes digital notes, textbooks, syllabus templates, and previous year examination papers (PYQs). Its core mission is to provide an aesthetic, zero-friction, incredibly fast study resource portal that remains completely accessible and durable through unique self-healing local-to-cloud server features.

2. Programming Languages & Core Technologies Used

Front-End Web UI

  • JavaScript (ES6+) / JSX: Powering the fully responsive single-page visual components.
  • React (v18.3.1): Manages interactive visual lifecycle hooks, layout state management, and real-time updates.
  • React Router DOM (v7.15.0): Coordinates paths to pages (e.g., modules, notes, PYQs, and administration views).
  • Tailwind CSS (v3.4.17): Handles the modern, eye-safe high-contrast dark visual aesthetics with custom layouts.
  • Framer Motion (v11.18.0): Hardware-accelerated UI transition effects (e.g., hover scaling, slide animations).
  • Lucide React: Standardized crisp vector icons.
  • Axios: Coordinates REST API connection with custom headers and error interceptors.

Back-End Platform

  • Node.js: The structural asynchronous execution runtime.
  • Express (v4.19.2): Handles the JSON routers, static asset delivery, upload stream processing, and server-side logic.
  • JSON Web Tokens (JWT): Signed bearer security headers representing admin sessions.
  • Bcrypt.js: Provides cryptographic database hashing for root accounts.

Database & File Persistence

  • MongoDB Atlas: Highly resilient NoSQL database client organizing complex records.
  • Hybrid Base64 Binary Stream Model: Files below 15MB are converted to base64 strings and written permanently to the database. If a container restart reboots local server storage, the files are dynamically restored from MongoDB and rewritten back to local storage automatically.

5. Back-End Express API Routings

Endpoint HTTP Method Access Control Functional Purpose

/api/subjects GET Public Lists all curriculum courses. Cache-optimized.

/api/files/:id/view GET Public Increments view counts and opens standard PDF data inline.
/api/files/:id/download GET Public Triggers attachment download and updates dashboard metrics.
/api/admin/files POST Admin (JWT Protected) Uploads documents, parsing binary streams into cloud DB fallback storage.

6. Highlighted System Capabilities & Workflows

A. Automated Disaster Recovery (Self-Healing Storage):
Most cloud deployments use ephemeral server instances. When a container reboots, its local file storage is completely erased. BITVERSE resolves this by acting as a hybrid database repository. When file uploads occur, the binary buffer is saved in MongoDB as base64 data. If a requested local file is missing, the backend instantly recreates the binary, updates local directories, and returns the file stream without interrupting the student's reading session.
B. Interactive Binary Search Tree (BST) visualizer:
A purpose-built algorithmic module tailored for First-Year engineering students studying computer science. It animates insertion, deletion, and search actions in real-time using interactive graphical nodes, compiling preorder, inorder, and postorder notations instantly.

7. Reproduction Guide (To Build from Scratch)

  1. Verify Node.js (v18+) and MongoDB are running locally or on MongoDB Atlas.
  2. Clone the repository and run npm install to install dependencies in both backend and frontend workspaces.
  3. Configure required parameters in a root .env file (including MONGODB_URI, JWT_SECRET, and ADMIN_EMAIL).
  4. Compile the React production distribution by executing npm run build.
  5. Launch the live, optimized digital ecosystem using npm start.
